## Поток Данных

### Голосовая Команда

```
[User Voice] → [Telegram] → [VoiceProcessor (Vosk)] 
    → [CommandRouter] → [Handler] → [ContextManager]
    → [EmotionEngine] → [LLM (Groq)] → [TTS (Silero)]
    → [TimingAnalyzer] → [Voice Response]
```

### Текстовая Команда

```
[User Text] → [Telegram] → [CommandRouter] → [Handler]
    → [ContextManager] → [Response] → [Optional TTS]
```

## Модули

### Core (Ядро)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `telegram_bot.py` | Telegram интерфейс | aiogram 3 |
| `voice_processor.py` | Распознавание речи | Vosk |
| `command_router.py` | Маршрутизация команд | Groq API |
| `context_manager.py` | Управление контекстом | Custom |
| `timing_analyzer.py` | Анализ производительности | Custom |

### Browser (Браузер)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `yandex_controller.py` | Управление Яндекс | Selenium, CDP |

### Windows (Окна)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `window_manager.py` | Управление окнами | pygetwindow, pyautogui |

### TTS (Голос)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `tts_engine.py` | Синтез речи | Silero TTS |

### Emotions (Эмоции)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `emotion_engine.py` | Эмоциональный интеллект | Custom + librosa |

### Reminders (Напоминания)

| Модуль | Описание | Технологии |
|--------|----------|------------|
| `reminder_system.py` | Таймеры, будильники | schedule |

## Конфигурация

```
config.py
├── Paths (BASE_DIR, CACHE_DIR, DATA_DIR)
├── Assistant Info (VA_NAME, VA_VER, VA_ALIAS)
├── Telegram (TELEGRAM_BOT_TOKEN, TELEGRAM_WHITE_LIST)
├── API Keys (PICOVOICE_TOKEN, GROQ_API_KEY)
├── TTS (TTS_CACHE_ENABLED, TTS_STREAMING_ENABLED)
├── Context (CONTEXT_MAX_COMMANDS, CONTEXT_TTL)
└── Emotions (EMOTION_DETECTION_ENABLED)
```

## Безопасность

- **White List** — только авторизованные пользователи
- **API Keys** — в отдельном файле dev.env
- **No SQL Injection** — используем параметризованные запросы
- **Input Validation** — проверка всех входных данных

## Масштабируемость

### Текущая Архитектура
- Однопользовательская (Telegram бот)
- Локальное выполнение команд
- Синхронная обработка

### Возможные Улучшения
1. **Многопользовательский режим** — разделение контекстов
2. **Микросервисы** — разделение на независимые сервисы
3. **Очереди** — Redis/RabbitMQ для масштабирования
4. **Кластеризация** — несколько инстансов бота
